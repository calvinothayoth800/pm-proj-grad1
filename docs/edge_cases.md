# AI & API Resilience Rules & Edge Cases

This document maps out the system constraints, API limitations, and behavioral edge cases that have been hardened in the playlist curation and editing pipelines.

---

## 1. API Resilience & Limits

### 1.1 Spotify `/v1/artists` 403 Forbidden Bypass
- **The Issue**: Spotify restricted access to the `/v1/artists` and `/v1/artists/{id}` endpoints for Client Credentials developer tokens, returning a `403 Forbidden` error. This made direct artist genre retrieval impossible.
- **The Guard**: Bypassed using `classifyTracksWithGroq`, a batch LLM classifier that maps tracks to genres and features (e.g. `["lofi", "beats", "rap", "vocals"]`) using Llama 3.1 8B's music catalog knowledge. This runs in under 300ms, avoids external API lookups, and keeps semantic filters working.

### 1.2 Spotify Rate Limits (429 Errors)
- **The Guard**: Set a conservative ceiling of `80` candidate recommendations per request and batch-process all frontend/backend operations. Added try-catch fallbacks to default objects for Groq rate limits.

## 2. Curation & Generation Nuances

### 2.1 Deduplication & Flexible Sizing (`existingTrackIds`)
- **The Issue**: If the user asked to `"add 3 lofi songs"`, the system would generate 3 tracks, but if 2 of them were already present in the playlist, the client-side deduplication filter would drop them, leaving only 1 newly added track.
- **The Guard**: The client passes the current playlist track IDs to `/api/curate` via `existingTrackIds`. The curation engine filters out these IDs from the search/recommendation candidate pool *before* returning the final list, ensuring the client receives exactly the requested number of unique tracks.

### 2.2 Exclusions vs. Explicit User Intent (Dynamic Overrides)
- **The Issue**: When curating a study lofi beats playlist, the system automatically appends general exclusions like `"rap"`, `"vocals"`, and `"pop"`. If a user explicitly requested `"add a lofi rap song"`, these general exclusions would block the requested track from appearing.
- **The Guard**: Implemented dynamic overrides in `/api/curate`. If the prompt explicitly contains a genre keyword (like `"rap"`, `"pop"`, `"vocals"`, or `"edm"`), the curation engine filters out that keyword from the exclusions list, honoring the user's query.

### 2.3 Priority Queries for Specific Artists/Tracks
- **The Issue**: Standard semantic search queries generated by the LLM are broad. If the user asked for a specific artist (e.g. `"Nujabes"`) or track (e.g. `"Wutang 4-eva"`), broad queries might fail to locate them in the top catalog results.
- **The Guard**: Added target parsing schema (`target_artist`, `target_track`) to the Groq curation parser. Priority queries in the form of `artist:"Nujabes"` or `track:"..."` are constructed and injected at the absolute top of the semantic search queue.

### 2.4 Candidate Pool Shuffling for High Diversity
- **The Issue**: Repeatedly running the same prompt (e.g. `"curate chill beats"`) could return the exact same tracklist if popularity scores were identical.
- **The Guard**: Candidate tracks are shuffled using a Fisher-Yates algorithm (`shuffleItems`) before being processed by the tiered fallback pipeline, ensuring high diversity and fresh song recommendations on every run.

## 3. Editing & Modifier Nuances

### 3.1 Pure Additive Commands Guard
- **The Issue**: When the user entered a purely additive command (e.g. `"add 1 rap song"`), the LLM agent would sometimes aggressively recommend removing current lofi tracks to "clean" the playlist.
- **The Guard**: Implemented a code-level guard in `/api/playlist-agent`. If the command contains addition keywords (e.g. `add`, `fill`, `expand`) and contains no removal keywords (e.g. `remove`, `delete`, `no`), the backend forces `remove_track_ids = []`, blocking accidental removals.

### 3.2 Dual-Sided Hybrid Filter (Exclusion + Protection Whitelists)
- **The Issue**: Repeatedly running `"remove non-lofi songs"` on a clean playlist could lead to over-filtering. The LLM might flag lofi tracks like Kudasaibeats (due to vocal chop tags) or Elijah Who (due to electronic tags).
- **The Guard**: The hybrid filter checks track genres against a **protection whitelist** (preserving tracks containing `lofi`, `chillhop`, `beats`, `ambient`, or `study` without `rap` tags) and deletes protected track IDs from the removal plan.

### 3.3 Catastrophic Deletion Guard
- **The Guard**: Enforces a strict removal ceiling. The agent can never delete more than 80% (rounded down) of the current playlist, preventing empty-state UI errors.
